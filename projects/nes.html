NES — Novel Encryption Standard: Lightweight Block Cipher

A lightweight block cipher designed for IoT and edge device encryption with formal security proofs.

Submitted to IEEE TIFS Python · C · SageMath · LaTeX

Overview

NES (Novel Encryption Standard) is a lightweight block cipher designed for resource-constrained environments such as IoT devices and edge computing nodes. Developed by Nilotpal Guha, Hiranmoy Roy, and Debotosh Bhattacharjee, NES has been submitted to IEEE Transactions on Information Forensics and Security (TIFS) for publication.

As IoT devices proliferate, the need for lightweight cryptographic primitives that can operate efficiently on devices with limited processing power, memory, and energy becomes critical. NES addresses this by providing strong security guarantees while maintaining minimal hardware footprint and computational overhead.

Key Features

  • 64-bit Block Size with 128-bit Key

    Balances security strength with efficiency, providing adequate block size for practical applications while keeping key management manageable for constrained devices.

  • Hardware Efficiency

    Optimized for low gate equivalent (GE) count, enabling implementation on resource-constrained hardware with minimal area overhead.

  • Strong Diffusion Properties

    Formally proven resistance against differential and linear cryptanalysis, the two most fundamental attacks on block ciphers.

  • Side-Channel Resistance

    Evaluated for resistance against power analysis and timing attacks, with design considerations that minimize information leakage through side channels.

  • Python Reference Implementation

    Complete Python reference implementation for benchmarking, testing, and integration into larger systems.

  • Full Academic Paper

    Comprehensive paper with formal security proofs, implementation details, and comparison with existing lightweight ciphers like PRESENT, GIFT, and SKINNY.

Design Philosophy

NES follows a substitution-permutation network (SPN) structure, optimized through careful analysis of diffusion layers and S-box design. The cipher uses a combination of bit permutation and modular addition operations that are efficient in both hardware and software implementations. The key schedule is designed to provide adequate security against related-key attacks while remaining computationally lightweight. Formal security proofs are provided using the provable security framework, establishing bounds on the advantage of any practical adversary.
