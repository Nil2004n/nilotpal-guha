TrinityOS — Custom Offensive Security Operating System

A custom-built Debian-based Linux distribution for offensive security professionals.

Active Development Debian · Bash · Python · live-build

Overview

TrinityOS is a custom-built, Debian-based Linux distribution designed for offensive security professionals and penetration testers. Built from the ground up by Nilotpal Guha, TrinityOS packages a carefully curated toolset for reconnaissance, exploitation, post-exploitation, and reporting into a reproducible bootable ISO.

Unlike generic security distributions, TrinityOS is purpose-built with a reproducible build system, ensuring every ISO is identical and verifiable. The distribution emphasizes a terminal-centric workflow with custom Xfce configuration, enabling operators to work efficiently without unnecessary GUI overhead.

Key Features

  • Debian Stable Base

    Built on Debian stable with a custom-hardened kernel configuration, ensuring reliability and security at the OS level.

  • Pre-installed Pentest Toolset

    Includes Metasploit Framework, Burp Suite, Nmap, BloodHound, Impacket, John the Ripper, Hashcat, and 100+ other tools pre-configured and ready to use.

  • Custom Recon & Exploit Scripts

    Bash and Python scripts for automated reconnaissance, service enumeration, and exploit delivery, reducing repetitive manual tasks.

  • Reproducible Build System

    Uses Debian live-build for fully reproducible ISO generation. Every build can be verified against the source repository.

  • LUKS Full-Disk Encryption

    Optional full-disk encryption with LUKS, ensuring sensitive tooling and data remain protected at rest.

  • Terminal-Centric Xfce Desktop

    Custom Xfce desktop environment optimized for terminal-heavy workflows with tiled window management and custom keybindings.

Tech Stack

Debian Linux Kernel Bash Python3 live-build Xfce systemd squashfs

Architecture

TrinityOS follows a layered architecture built on top of Debian's live-build system. The base layer provides a minimal Debian installation with kernel hardening applied. The tool layer adds pre-configured security tools with persistent configuration support. The automation layer includes custom scripts for common pentest workflows. The desktop layer provides a customized Xfce environment with terminal-centric keybindings and layouts.

PINACA — Parallel Intrusion Network Analysis & Correlation Architecture

Distributed intrusion detection with real-time threat correlation across parallel sensor nodes.

Research Prototype Python · C · eBPF · Elasticsearch

Overview

PINACA (Parallel Intrusion Network Analysis & Correlation Architecture) is a distributed intrusion detection system designed for real-time network monitoring and threat correlation across multiple sensor nodes. Developed by Nilotpal Guha, PINACA addresses the challenge of correlating alerts across segmented networks without centralized bottlenecks.

Traditional IDS deployments struggle with scale — as network segments grow, centralized collectors become bottlenecks and single points of failure. PINACA solves this by distributing the analysis load across sensor nodes while maintaining a correlation engine that can link events across the entire network topology.

Key Features

  • Distributed Sensor Architecture

    Deploy sensors across network segments with a centralized correlation engine that aggregates alerts without introducing latency or single points of failure.

  • Real-Time Packet Inspection

    Uses AF_PACKET and eBPF for high-performance packet capture and analysis at line rate, enabling deep packet inspection without impacting network throughput.

  • Custom Threat Correlation Rules

    Rule engine that correlates events across multiple sensors, detecting multi-stage attacks that span network segments — something single-sensor IDS solutions miss.

  • Elasticsearch + Kibana Dashboard

    Pre-configured ELK stack integration for real-time visualization, alerting, and historical analysis of network threat data.

  • REST API for SOAR Integration

    Full REST API enables integration with Security Orchestration, Automation, and Response (SOAR) platforms for automated incident response workflows.

  • Python Plugin System

    Extensible plugin architecture allows custom detection rules to be written in Python, enabling rapid development of organization-specific threat signatures.

Tech Stack

Python3 C eBPF AF_PACKET Elasticsearch Kibana Redis Docker

Architecture

PINACA operates on a three-tier architecture: sensor nodes perform local packet capture and analysis using eBPF programs for kernel-level performance. An aggregation layer collects and normalizes sensor outputs via Redis message queues. The correlation engine applies cross-sensor rules to detect distributed attack patterns, feeding results into Elasticsearch for storage and Kibana for visualization. The entire stack is containerized with Docker for easy deployment across heterogeneous environments.

Meghnad — AI-Powered Vulnerability Scanner

Machine learning-powered vulnerability scanning with intelligent exploitability prioritization.

Beta Python · Scikit-learn · asyncio · Shodan

Overview

Meghnad is an intelligent vulnerability scanner that leverages machine learning to prioritize findings by exploitability and reduce false positive noise. Built by Nilotpal Guha, Meghnad goes beyond signature-based scanning by correlating CVE data with real-world exploit availability and network context.

Traditional vulnerability scanners overwhelm security teams with hundreds of low-severity findings. Meghnad changes this by applying ML models trained on historical CVE data and exploit databases to score each finding by actual exploitability, helping teams focus on what matters most.

Key Features

  • ML-Based Exploitability Scoring

    Uses CVE data and the Exploit Prediction Scoring System (EPSS) to assign realistic exploitability scores to each finding, prioritizing vulnerabilities that are most likely to be exploited in the wild.

  • Parallel Scanning Engine

    Async I/O-based scanning engine that can evaluate thousands of hosts simultaneously, dramatically reducing scan completion times compared to sequential scanners.

  • Custom Plugin Architecture

    Modular plugin system allows custom scan modules to be written in Python, enabling organization-specific vulnerability checks alongside standard CVE scanning.

  • Shodan & Censys Integration

    Automatically discovers external attack surface by querying Shodan and Censys APIs, mapping discovered services back to internal assets.

  • Report Generation

    Generates detailed reports in HTML and JSON formats with executive summaries, technical findings, and remediation guidance.

  • CI/CD REST API

    Fully documented REST API enables integration into CI/CD pipelines for automated security gate checks in deployment workflows.

Tech Stack

Python3 Scikit-learn asyncio aiohttp Shodan API Censys API Jinja2

Architecture

Meghnad's architecture separates scanning, analysis, and reporting into distinct layers. The scanning layer uses asyncio for concurrent host evaluation. The analysis layer applies ML models to raw findings, correlating them with EPSS scores and exploit database data. The reporting layer uses Jinja2 templates to generate human-readable HTML reports and machine-parsable JSON output. The Shodan/Censys integration runs as an optional surface discovery module that feeds host targets into the scanning pipeline.

Kalki — Autonomous Penetration Testing Framework

End-to-end autonomous pentesting from recon to privilege escalation in a single pipeline.

Alpha Python · Rust · Impacket · BloodHound

Overview

Kalki is an autonomous penetration testing framework that chains together reconnaissance, vulnerability discovery, exploitation, and privilege escalation into a single automated pipeline. Built by Nilotpal Guha, Kalki is designed to augment red team operations by automating repetitive attack chains while leaving strategic decisions to the operator.

Named after the prophesied final avatar of Vishnu who restores order, Kalki automates the systematic dismantling of network defenses — from initial foothold to domain administrator — while maintaining full operator visibility and control over every stage of the attack.

Key Features

  • Autonomous Attack Chain Orchestration

    Automatically chains reconnaissance, exploitation, and post-exploitation phases, discovering and executing optimal attack paths without manual intervention at each step.

  • Modular Exploit Plugin System

    Extensible plugin architecture for exploits and post-exploitation modules, allowing custom attack capabilities to be added without modifying the core framework.

  • Active Directory Attack Automation

    Native support for Kerberoasting, AS-REP roasting, DCSync, golden ticket attacks, and other AD-specific attack techniques with automatic discovery and execution.

  • Automatic Credential Harvesting

    Extracts and consolidates credentials from compromised hosts, automatically attempting credential reuse across the network for lateral movement.

  • Real-Time Operator Console

    WebSocket-based console provides real-time visibility into attack progress with session management, allowing operators to intervene or redirect attacks as needed.

  • MITRE ATT&CK Report Generation

    Generates comprehensive reports with findings mapped to MITRE ATT&CK techniques, providing actionable intelligence for both red and blue teams.

Tech Stack

Python3 Impacket Scapy BloodHound Rust WebSocket

Architecture

Kalki operates as a pipeline orchestrator with four main stages: Recon, Exploit, Post-Exploit, and Report. The Recon stage uses BloodHound and custom LDAP queries to map Active Directory attack paths. The Exploit stage applies vulnerability findings to establish initial access. The Post-Exploit stage harvests credentials and performs lateral movement. The Report stage generates MITRE ATT&CK-mapped documentation. Rust modules handle performance-critical operations like packet crafting and credential parsing, while Python manages the high-level orchestration and operator console.

Iskabon — High-Performance Packet Capture Engine

Wire-speed packet capture for 10Gbps+ networks using DPDK and zero-copy architecture.

Release Candidate C · DPDK · libpcap · BPF · Prometheus

Overview

Iskabon is a high-performance packet capture and analysis engine designed for 10Gbps+ network environments. Built by Nilotpal Guha, Iskabon uses DPDK (Data Plane Development Kit) and zero-copy architecture to achieve wire-speed packet capture without dropping frames, addressing the limitations of traditional tools like tcpdump in high-throughput scenarios.

At 10Gbps, a standard Ethernet frame arrives every 6.72 nanoseconds. Traditional packet capture tools built on libpcap cannot keep up with this rate without dropping packets. Iskabon bypasses the kernel networking stack entirely using DPDK, writing captured packets directly to user-space memory through zero-copy ring buffers.

Key Features

  • DPDK Zero-Copy Capture

    Bypasses the kernel networking stack entirely, capturing packets directly from NIC hardware queues into user-space memory through DPDK's zero-copy mechanism.

  • Multi-Speed Support

    Supports 10Gbps, 25Gbps, and 40Gbps network interfaces with automatic speed detection and optimal buffer configuration for each rate.

  • Lock-Free Ring Buffer

    Custom SPSC (Single Producer, Single Consumer) ring buffer implementation that eliminates lock contention between the capture thread and analysis threads.

  • PCAP & PCAP-NG Output

    Captured packets are written in standard PCAP and PCAP-NG formats, ensuring compatibility with Wireshark, tcpdump, and other analysis tools.

  • BPF Packet Filtering

    Real-time packet filtering using standard BPF (Berkeley Packet Filter) expressions, allowing operators to capture only relevant traffic at line rate.

  • Prometheus Metrics

    Built-in Prometheus metrics endpoint exposes capture statistics including packet rates, drop counts, buffer utilization, and CPU usage for monitoring integration.

Tech Stack

C DPDK libpcap BPF Prometheus client Makefile

Architecture

Iskabon's architecture is built around DPDK's poll-mode driver model. A dedicated capture thread polls NIC hardware queues directly, bypassing the kernel entirely. Captured mbufs (memory buffers) are placed into a custom lock-free SPSC ring buffer. Analysis threads consume from this ring, applying BPF filters and writing to PCAP output files. The Prometheus metrics thread periodically samples capture statistics and exposes them on an HTTP endpoint. This architecture ensures zero packet loss at line rate while maintaining minimal CPU overhead through busy-polling.
